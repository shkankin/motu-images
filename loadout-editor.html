How this works
The MOTU Vault app reads loadouts.json at the repo root on every sync. Schema: 
{
  "version": 1,
  "loadouts": {
    "he-man-1982": ["Power Sword", "Half Sword", "Shield"],
    "skeletor-1982": ["Havoc Staff", "Power Sword (half)"]
  }
}
Each user's local override (managed in-app via the Edit Loadout button on the accessory picker) takes priority over the repo entry for the same figure ID. Use this editor to seed defaults for figures you've already curated locally, then ship them to everyone.

localStorage is per-origin. The "Seed from this device" button only sees your in-app local edits if this editor is opened from the same domain as the deployed app (e.g. shkankin.github.io). If you open this file via file://, you can still build a list and export the JSON. To pull your existing local edits when opened standalone, run this in the deployed app's browser console: copy(localStorage.getItem('motu-acc-avail')), then paste into the textarea above and click "Load from textarea".

v6.04 plan: the schema is intentionally a flat array of strings today. A future bump may extend entries to {name, required} objects — string entries will be auto-treated as required: true.
